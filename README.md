# encounter.bio

Structural reads for immunotherapy trial design. Built on Generative Geometry (van der Klein, 2026).

Two report types:

- **Retrospective reports** — re-analyses of published immunotherapy trials. Free with a work email. The form unlocks all retrospective reports for 90 days via cookie.
- **Prospective reports** — forecasts for trials in design. Contact form only; no automated download. Raimo replies personally to discuss scope and pricing.

## Stack

Flask + Jinja, deployed to Render. No database. Leads written to CSV files. Notification emails sent via SMTP (configure via Render env vars; falls back to stdout logging if no SMTP is configured).

## Local development

```bash
pip install -r requirements.txt
python app.py
# → http://localhost:5000
```

## Deploy to Render

1. Push this repo to GitHub.
2. In Render: New → Web Service → connect the repo.
3. Render reads `render.yaml` and configures the service automatically.
4. Set the SMTP environment variables in the Render dashboard:
   - `SMTP_HOST`, `SMTP_PORT`, `SMTP_USER`, `SMTP_PASS`, `SMTP_FROM`, `NOTIFY_TO`
   - If unset, notifications are logged to stdout instead of sent.
5. Point `encounter.bio` DNS to the Render service.

## Pipeline integration

The prose in `data/reports.json` (deck copy, headlines, narratives) is hand-curated. The numeric predictions behind that prose come from the pipeline at `pipeline/`. The pipeline reads no website files; the website reads no pipeline files at request time. The two are linked at editing time:

```bash
make pipeline-all      # rebuild predictions from formula.predict_orr against 183 calibrated trials
make pipeline-sync     # rebuild + run check_drift.py to flag drift between reports.json and pipeline output
```

`pipeline-sync` produces `pipeline/data/predictions_summary.json` (one record per trial with the framework's current direction, predicted_value, falsifier, layer ORRs, method trace) and then runs `check_drift.py`, which compares it against `data/reports.json` slug-by-slug and warns about any mismatches. Run this before publishing to make sure the prose hasn't drifted from the framework numbers.

The pipeline can also be open-sourced separately later — its only dependency on the website is the slug mapping in `build_predictions_summary.py`.

## Adding a new retrospective report

1. Add an entry to `data/reports.json` with `type: "retrospective"`, slug, title, deck, indication, drug, n, date.
2. Create `templates/reports/<slug>.html` extending `base.html`. Wrap the body in `{% if not unlocked %}<form>{% else %}<full content>{% endif %}` so the gate works.
3. Commit and push. Render auto-deploys.

The home page reads `reports.json` at request time, so no code change is needed beyond the new template.

## Adding a new prospective report

Same as retrospective, but `type: "prospective"` in `reports.json` and include a `prediction` field with the headline forecast (shown in the prediction inset on the card). Use the contact form (`/lead-prospective`) inside the report template instead of the email gate. Create `templates/reports/<slug>.html` accordingly.

## Brand

- **Colors**: white background, orange accent (`#c84a2e`), grayscale ink. Green (`#2d7a3e`) is reserved for empirical signals (responders, confirmed predictions) and never for UI affirmation.
- **Type**: Charter serif for body and headlines, system sans for UI labels and table headers, JetBrains Mono for numbers and patient IDs.
- All values defined as CSS variables in `static/brand.css`. Every template imports it.

## Email gate

Retrospective reports use a four-field form (work email, name, organization, optional purpose). Free email providers (gmail, outlook, etc.) are blocked at the form level with a friendly error and an escape hatch to email directly. On successful submit, the server sets a 90-day cookie (`encounter_member`) that unlocks every retrospective report on the site. No accounts, no login, no database.

## Prospective gate

Prospective reports never auto-unlock. The contact form posts to `/lead-prospective`, the lead is recorded to CSV, a notification email is sent, and the user sees a thanks page explaining that Raimo will reply within two business days. The price-discovery question happens in that reply, not on the site.
