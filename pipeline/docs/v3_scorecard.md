## Errata to v1 (10.5281/zenodo.19408510) and v2 (10.5281/zenodo.19432232)

v1 and v2 of this scorecard contained material errors that v3 corrects:

1. **Date drift on confirmed hits.** v1/v2 reported KeyVibe-006, LATIFY, TiNivo-2, and STAR-221 as 'pre-publication confirmations' with dates that were off by 4–15 months. In every case, the trial had already publicly read out before v1 was published. These are **backtested directional hits**, not prospective wins. v3 reframes them as such.

2. **Prediction provenance.** The numerical predictions in v1/v2 were generated by Claude reasoning about the framework against trial designs in conversation, without invoking the framework's prediction engine (`formula.predict_orr`) and without persisting any audit trail. v3 numbers come from running `formula.predict_orr` against a calibration table back-calculated from 183 historical trials with observed ORRs, persisted to a database that anyone with the codebase can re-run.

3. **Trial metadata corrections.** AdvanTIG-302 was misclassified as cervical cancer in v1/v2 — it is 1L NSCLC (NCT04746924). KEYLYNK-010 was cited as an NSCLC PARP+ICI precedent — it is mCRPC (NCT03834519). VOLGA n was cited as 830 — actual is 677. Tagmokitug indications were listed as pancreatic/MSS CRC — actual is HNSCC/CRC/gastric/esophageal.

4. **VOLGA safety run-in addressed.** v1/v2 predicted pCR ≥50% for VOLGA's triplet but did not engage with the publicly reported 35% pCR (n=17) from the safety run-in at ESMO 2024. v3's calibrated framework prediction is 40.6% pCR — closer to the run-in than v1/v2 but still above it; this is flagged honestly as a possible framework overestimation.

v1 and v2 are not deleted but are superseded by v3. Anyone landing on the concept DOI should be directed to the most recent version.

## Methodology

Predictions are generated by `formula.predict_orr(domain, combo, stage)` from the Generative Geometry framework's single source of truth (`formula.py` v13). The function takes:

- a **disease domain** (urothelial, melanoma, NSCLC, HCC, gastric, SCLC, etc.)
- a **drug combo** (an unordered set of agent IDs joined by `+`)
- a **treatment stage** (neoadjuvant, adjuvant, metastatic 1L/2L/3L+)

And returns a predicted ORR with one of four method tiers:

1. **Calibrated** — exact (domain, combo, stage) match in the calibration table back-calculated from 183 historical trials with observed ORRs.
2. **Stage-scaled** — same combo at a different stage; M transferred via the agent type model.
3. **Single-scaled** — single agent calibrated against any stage.
4. **Novel from components** — agent layer split (L3 immune, L4 molecular), Bliss independence within layer, cross-layer Bliss combine, clamped at the stage's depth resistance ceiling.

For neoadjuvant trials, ORR is translated to pCR using the canonical ratio observed in EV-103 cohort H, NIAGARA, and KEYNOTE-B15 (pCR ≈ 0.55 × ORR for urothelial neoadjuvant; lower for gastric and NSCLC).

## Methodology limits

The following are known gaps in the current implementation:

1. **Acquired resistance not modelled natively.** The base formula has no `prior_line_failure` modifier. For IMbrave251 (atezo continuation after atezo+bev failure), the raw `predict_orr` output is +15.4pp (WIN) because it treats the 2L combo as a novel structural combination. The final FAIL call comes from a class precedent override layered on top of the formula output, citing the IO-progression-after-progression literature. v4 should move this override into the formula itself via an explicit `prior_line_failure` penalty rather than handling it in the pipeline wrapper.

2. **Depth-2 ceiling clamping.** For neoadjuvant indications at depth 2, most cross-layer immune+molecular combinations saturate against the depth resistance ceiling (~70–74%). Differentials must be read at the per-layer level, not the final clamped output.

3. **ORR-to-pCR translation is fixed-ratio.** The 0.55 ratio for urothelial neoadjuvant is anchored to three trials. Trials testing novel combos may have different ORR-to-pCR ratios.

4. **Calibration table scrubbing required.** Two evidence rows in the GG framework's `systems.db` (IMpower010, PEARLS/KEYNOTE-091) report `observed_orr=0` because their actual endpoint was DFS, not ORR. v3 uses a `safe_cal_table()` wrapper that filters them out.

5. **Backtested predictions are class-level, not formula-derived.** The four backtested hits are scored as `method='backtested'` because they were generated from class-level structural arguments rather than from `formula.predict_orr`. They could be re-derived through the formula if their drug agents were added to the relevant disease domains.

## How to reproduce

The encounter pipeline expects access to `formula.py` (v13) and the GG framework's `systems.db` calibration database.

```
python3 seed_trials.py             # populate trial schema
python3 seed_dellphi312.py         # add DeLLphi-312
python3 seed_backtested.py         # add backtested historical trials and readouts
python3 make_predictions_all.py    # run formula.predict_orr against all trials
python3 score_predictions.py       # score predictions vs readouts
python3 export_v3.py               # regenerate this document
```

Every prediction in `encounter.db` has a `method_trace` field with the full output of `formula.predict_orr` including the calibration table size, agent substitutions, per-layer ORRs, depth ceiling, and prediction date.
